Features

What it does

Stash watches the clipboard and saves things on its own. The rest is there to help you keep it all in order.

▣

Saves things on its own

It watches the clipboard and saves new text, links, and images as they show up. It hashes what it saves so you don’t end up with duplicates, and it ignores its own copies. You can also paste, type a note, or just drag files straight in.

⌘ V paste & save ⌘ ↵ save a note
☷

Folders and tags

Drag a card onto a folder to move it there. Add as many tags as you want, and pin the things you keep reaching for.

↗

Links get a preview

When you’re online, links pull in a thumbnail and the page title. Offline, every link still gets a little generated favicon.

✓

To‑do lists, properly

Make as many as you like. A task can have a due date, notes, and subtasks. Overdue ones turn red, and unfinished tasks sort to the top by whatever’s due soonest. “Clear done” sweeps up the rest.

⚲

Find anything fast

One search runs across the text, link URLs, file names, tags, and folder names, over the whole collection rather than just what’s on screen. Sort by newest or oldest; pinned stays on top.

Where your stuff lives

It all stays on your Mac.

Everything Stash saves goes into a SQLite database on your computer. Images and files are written to disk, and the database just keeps the path to them, not the bytes. The only time it touches the network is to fetch link previews from the sites you save.

  • ✓
    A local SQLite database holds the whole collection. No account, no sign‑in, no cloud.
  • ✓
    Nothing gets sent anywhere, apart from those link‑preview fetches.
  • ✓
    It updates itself: checks for a newer version, verifies the signature against a key built into the app, and installs it, so you don’t re‑download the dmg.
  • ✓
    The whole thing is open source. Rust backend, plain HTML/CSS/JS frontend, all in one repo.

Under the hood

How it actually works

Built with Tauri v2: a Rust backend and a plain HTML/CSS/JS frontend with no build step.

01

The clipboard watcher

A background thread polls the clipboard with arboard and hashes new contents with sha2, so the same thing never gets saved twice and Stash skips its own copies.

02

Local storage

Everything goes through rusqlite (bundled) into a SQLite database in WAL mode. Image and file blobs live on disk; the database keeps the path, not the bytes.

03

Link previews

Previews are fetched in Rust with reqwest over rustls to dodge CORS, pulling the og:image and page title. Favicons are generated client‑side as SVG.

04

Signed updates

On launch it checks for a newer release, downloads it, verifies the signature against a public key built into the app, installs it, and restarts. No re‑downloading the dmg.

Installing

It takes about a minute.

Download the dmg

Grab Stash_universal.dmg from the latest release, open it, and drag Stash into your Applications folder.

Open it the first time

Right‑click Stash and choose Open, then Open again. macOS warns you here because the app isn’t signed with a paid developer account.

If it says “damaged”

Run this in Terminal once, then open it again:

$xattr -cr /Applications/Stash.app
